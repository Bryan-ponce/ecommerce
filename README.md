## <ins>**Objetivo**</ins>

El sistema de gestión e commerce que seleccione tiene como finalidad administrar productos, usuarios y pedidos de manera eficiente generando confianza en los usuarios, garantizando una interacción eficaz entre tienda y cliente o usuario. Ofreciendo una experiencia amigable en el comercio digital.

## <ins>**Funcionalidades principales**</ins> 

Mi proyecto está desarrollado en lenguaje Goland y se encuentra estructurado en módulos independientes los cuales nos permiten mantener una estructura clara, concisa y escalable:

- **Módulo de carrito de compras:** permite la selección de productos por parte del cliente, cálculo automático del total y vinculación del carrito de compras con el usuario.

- **Módulo de gestión de pedidos:** este genera y confirma la selección de pedidos, con historial asociado al cliente.

- **Módulo de gestión de productos:** nos permite crear, listar, actualizar y eliminar productos garantizando un control eficiente del catalogo disponible en la tienda.

- **Módulo de gestión de reportes:** detalla las ventas realizadas, productos más vendidos e información consolidada sobre las ventas.

- **Módulo de gestión de usuarios:** este incluye el registro, configuración y eliminación de usuarios, incluyendo la asignación de roles y usuarios privilegiados dentro del sistema.

- **Módulo de historial de compras por usuario:** este permite consultar todos los pedidos realizados por un cliente específico. Con ella se genera un registro detallado de las compras.

- **Módulo resumen de ventas:** Este módulo ofrece un reporte filtrado por día, mostrando el total de ventas realizadas en una fecha determinada.

- **Módulo de formas de pago:** expone las opciones disponibles para completar una transacción: efectivo, transferencia bancaria o tarjeta, ampliando la experiencia del cliente.

**8.	Formas de pago:** opciones de pago disponibles para el producto
